# 2026.196.1135 — pr-validator: ARCHITECTURE GATE per-item trace in the POSTED comment + ZERO-ALIASES hook floor + alias-files-no-exception

The plugins leg of the guidance-level fix (B2 leg 1) for the RCA finding (task #49)
that a fresh `pr-validator` let an alias/boundary violation pass: the sdk#54
round-1 validator posted a PASS comment that OMITTED the `placement:` verdict and
carried NO per-item trace for the staged `charly/*.go` alias. The rules ALREADY
existed in item 15 (mechanical sub-checks + VERDICT DUTY); the DISCIPLINE gap was
that the POSTED comment did not carry them. This cutover closes that gap at the
guidance layer — for now AND the future.

## Strengthening — `internals/agents/pr-validator.md` item 15

- **PER-ITEM TRACE (mandatory in the POSTED comment).** New bullet: for each
  staged `charly/*.go` change, the POSTED validation comment carries an explicit
  line per item — any new/grown alias file (→ blocker); any new kit import (→
  blocker unless the same-PR body-move-out + per-site `until-K<n>` inventory
  conditions are VERIFIED and STATED); and the boundary-law placement (E/M/B/D/R)
  for new or moved core code, distinguishing a GENERIC kind-AGNOSTIC host
  mechanism (the four in-core M's — plugin loading, prescan-dispatch, the
  kind-decode materialize, the wire broker — or a class-generic `HostBuild` KIND
  seam such as `overlay`/`cli`/`step-emit`, which is permitted) from a
  PER-CAPABILITY seam (a provider WORD on the API surface, a `spec.<Kind>`
  field-read, a kind-word `switch`, a per-kind Go map — an R-item that LEAKED into
  core = blocker). A POSTED comment with NO per-item trace for a `charly/*.go` diff
  is an INCOMPLETE validation — the exact gap that let the alias slip through
  unflagged.
- **Alias files have NO migration exception** — made explicit in the mechanical
  sub-checks: the residual-call-site exception covers a plain kit IMPORT/CALL
  only, NEVER an alias FORM (an alias is a mislocated call site; the fix is MOVING
  the consumer, never re-exporting).
- **Hook-as-floor.** The new `pre-commit-gate.sh` ZERO-ALIASES hook (super leg) is
  named as the mechanical floor under this check: a hook block is a HARD FAIL
  (never a workaround target); a hook ALLOW does NOT discharge the validator's
  judgment, because IMPORT-PURITY of a plain kit IMPORT/CALL is validator-judged,
  not hook-gated.
- **`*Legacy*` relocation smell.** A `*Legacy*`-named identifier or file introduced
  alongside a body move is flagged — a redesign that DROPS the "Legacy" name+shape
  is expected, not a kept legacy shape renamed in place.
- **VERDICT DUTY** clarified: the `placement:` verdict must be IN THE POSTED
  COMMENT (an internal-only report the operator cannot see does not discharge the
  duty).
- **Checklist line 15** extended with the per-item-trace field + the hook-floor +
  hook-ALLOW-does-not-discharge notes.
