# 2026.189.0438 — pr-validator: refuse forbidden-framing dodges, enforce CLAUDE.md rigorously

Hardens the `pr-validator` agent spec so it stops accepting the cheats CLAUDE.md
already forbids but the checklist did not name:

- **Top-line mandate rewritten to RIGOROUS, TOTAL enforcement** — ANY plausible
  CLAUDE.md violation is a FAIL, the burden is on the PR to prove compliance,
  "when in doubt, FAIL", and a merge is an assertion of full compliance.
- **New checklist item 8 "no forbidden-framing dodges"** (with a matching
  output-format line): a load-surfaced failure answered with "it passes on an
  idle / serial / single-bed / re-run" FAILS — a concurrency defect is invisible
  to a serial run, so idle-green proves nothing (the concurrency mandate); it
  needs a full R1 RCA from the actual error line + a root-cause fix (sync
  primitive / store-lock ceiling / bounded fan-out), never serialize-to-hide, a
  retry/sleep/timeout-bump, or a "flake/transient/environmental/load"
  dismissal. "Pre-existing / out of scope / unrelated / follow-up" as a dodge to
  land FAILS (R2). Every concurrency issue must carry a root-cause RCA, and a
  shared-state change's R10 gate must be the CONCURRENT roster, not a serial run.

Motivating incident: `charly#4` (P1) landed with five load-surfaced bed failures
classified "verify green on the idle host" — exactly the cheat now refused.
