`charly candy` is no longer a thin forwarder to a hidden `__candy` core command. candy/plugin-candy is a
COMPILED-IN command plugin that OWNS its ENTIRE logic — the set/add-{rpm,deb,pac,aur} grammar AND the
comment-preserving yaml.Node manifest mutation. The generic yaml utilities (SetByDotPath / MappingChild)
moved from charly core to sdk/kit/yaml.go (ONE copy, also used by `charly box set` / `box scaffold`).
Swept the claim-keyed source-map references across the layer, go, charly-mcp-cmd, new, reconcile, and
image skills: `charly/yaml_setter.go` → `sdk/kit/yaml.go`; the candy verbs (CandyCmd/CandyAddPkgCmd/
detectPkgSection/appendCandyPackages) → `candy/plugin-candy/command.go`.
