# 2026.194.1750 — agents skill: teammate context-lifecycle policy

Codifies the operator-directed teammate context-space management strategy in
`/charly-internals:agents` (a new "Teammate context lifecycle — reuse is TASK-SCOPED,
never cross-task" subsection under "Delegation IS fresh context", plus a pointer from
"Agent lifecycle hygiene"):

- A teammate's loaded context is an ASSET on its own task chain (PR fix rounds, its
  chain's next leg, rebases, same-unit re-scopes) and a LIABILITY on any other unit.
- Reuse is task-scoped: a landed teammate is STOPPED, never re-tasked with a different
  unit — a new unit gets a fresh teammate (or an explicit context/workspace clear,
  which is equivalent: the new unit starts from zero).
- Assignments are sized one-unit-chain-per-context-budget at spawn; durable artifacts
  (scoping maps, verdicts, handoff notes) are written to files before an agent stops,
  making stopping lossless; validators may stay alive only between the legs of ONE PR
  chain when the next leg is imminent.
- Mid-task context pressure: compact-and-continue on the same task; separable
  remainders hand off via the worktree + handoff package to a fresh successor.
