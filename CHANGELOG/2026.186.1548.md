# remove/deploy skills: `charly remove --purge` now also drops the `<name>-overlay` images

`charly remove --purge` previously removed only named volumes; it now additionally drops the deploy's
encrypted volumes AND the synthesized `<name>-overlay` images an `add_candy:` overlay build produced
(charly/commands.go purgeDeployArtifacts). This closes the overlay-image leak: every disposable pod
check bed tears down via `charly remove --purge` (the bed runner's default cleanup), which never
reached the pod substrate's `charly bundle del`-only overlay drop, so `<name>-overlay` images
accumulated. The authoritative `/charly-core:remove` skill and the deploy skill's volume-integration
mention are updated to the full purge (named + encrypted volumes + overlay images).
