# settings skill: rewrite to the compiled-in-command-owns-its-logic model

`charly settings` is no longer a thin forwarder to a hidden `__settings` core command. The settings
skill now documents the current architecture: candy/plugin-settings is a COMPILED-IN command plugin that
OWNS the get/set/list/reset/path grammar + output; the config subsystem (read/write
~/.config/charly/config.yml + the credential store + engine/runtime resolution) stays in core
(charly/runtime_config.go), reached over the generic `HostBuild("settings")` seam
(charly/host_build_settings.go, resolveSettingsGet for the get special-cases). Dispatch is in-proc via
dispatchInProcCommand's threaded reverse channel. Also corrects the config path (settings.yml → the
actual config.yml) in three places.
