# 2026.195.0924 — git-workflow: multi-committer main-advance handling (out-of-tree PRs)

`git-workflow` SKILL: new "Multi-committer main advances — out-of-tree PRs from
other committers" subsection. Makes explicit that the delta-re-gate /
update-branch / forward-gitlink discipline is main-advance-agnostic and applies to
ANY merge regardless of source (an out-of-tree PR from another committer landing
mid-plan is treated identically to an internal orchestrator merge). Three
additions: (1) proactive detection — fetch `origin/main` (and each submodule main)
before each PR-open / merge, not only the orchestrator's own-merge broadcast;
(2) divergent-lineage submodule bump — an external merge bumping a submodule to a
commit NOT in the feat's gitlink ancestry is RE-RDD'd on a disposable bed, never
blindly descendant-wins; (3) the orchestrator OWNS external-advance detection +
broadcasts the rebase over any main advance, internal or external. Documentation
only — no behavioral change. The CLAUDE.md mandate pointer lands on the
superproject leg.
