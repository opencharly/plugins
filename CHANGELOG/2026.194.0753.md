**Migration-pattern exception** (folded into the pr-validator sub-check (ii)): the no-new-kit-import rule permits
the residual-call-site import created when a mechanism's BODY moves OUT of core in the SAME PR (net core-LOC
negative), each residual site inventoried "until-K<n>". Rationale: shared mechanisms have staggered consumers;
forbidding the residual import outright forces seam-proliferation (v2-hostile) or blocks every mechanism move on
its LAST consumer (kills incremental migration). The exception is self-limiting — net-negative LOC + inventory
required, and P16's strict end-state gate still demands zero at program end. (The hook's declaration-form guard is
unaffected — it blocks alias FORMS, not plain imports.)
