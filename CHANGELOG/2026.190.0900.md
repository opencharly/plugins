# 2026.190.0900 — pr-validator: zero-tolerance SDD / CUE-source gate

Hardens the `pr-validator` agent's SDD check (item 12) into a dedicated,
100%-enforced, zero-benefit-of-the-doubt CUE-source gate. Where item 12 previously
said only "cue:gen no-op, no gen drift", it now enumerates the whole
CUE-single-source mandate the validator must POSITIVELY confirm against the diff,
any one of which is a hard FAIL:

- **a. Reproducibility** — `task cue:gen` is a NO-OP on the committed tree;
  `TestGenReproducible` (sdk `spec/`) + `TestPluginsGenReproducible` pass; a
  `schema/*.cue` / `compiled_plugins:` / `*.proto` edit whose `*_gen.go` was not
  regenerated in the same commit FAILS.
- **b. No hand-edited generated files** — every `Code generated … DO NOT EDIT`
  file (`cue_types_gen.go`, `vocab_gen.go`, `version_gen.go`, plugin
  `params/cue_types_gen.go`, `plugins_generated.go`, `*.pb.go`) is regenerated
  from source, never hand-touched.
- **c. Wire types CUE-sourced without exception** — a hand-written
  `sdk/spec/*_wire.go` or plugin `params` struct not generated from a
  `schema/*.cue` def FAILS (a wire type is a plain/discriminated struct that
  `gengotypes` generates; it never needs a disjunction).
- **d. No unjustified `@go(-)` / hand-written schema type** — outside the
  spike-proven set (`union_types.go`, `hand_state_types.go`, `charly_names.go`,
  `plugin.proto`, `Op.Kind()`), a new one demands a full RCA + a pasted
  `cue exp gengotypes` spike proving CUE cannot express it.
- **e. Per-plugin schema single-source** — a plugin with authored input ships its
  own `schema/*.cue` and decodes into the generated typed `params`, never a
  hand-parsed `map[string]any`.

Mandate lives in CLAUDE.md "Schema Driven Design (SDD)"; the detail the gate points
to is `/charly-internals:go` "Generation coverage". Agent-definition edit only; no
rule body restated, no identifier renamed.
