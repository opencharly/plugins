# clean skill: rewrite to the compiled-in-command-owns-its-logic model

`charly clean` is no longer a thin forwarder to a hidden `__clean` core command. The clean skill now
documents the current architecture: candy/plugin-clean is a COMPILED-IN command plugin that OWNS the
command (flag grammar, category orchestration, output, the local pkg/arch makepkg sweep). The SHARED
retention engine (image-tag / build-candy / check-run pruning — multi-caller: `charly box build`,
`charly check run`, `charly box list tags`) stays in core (`charly/retention.go`), reached over the
generic `HostBuild("retention")` seam. Dispatch is in-proc via `dispatchInProcCommand`'s threaded
reverse channel. Same "plugin owns the logic + generic seams for the core-coupled bits" doctrine the
vm + pod deploy plugins established.
