# charly-doctor skill: rewrite to the compiled-in-plugin-owns-the-report model

`charly doctor` is no longer a thin forwarder to a hidden `__doctor` core command. candy/plugin-doctor
is a COMPILED-IN command plugin (command:doctor) that OWNS the entire host-dependency report — check
list, verdicts, human/JSON formatting, and the binary/file probes it runs itself. It reaches ONLY the
genuine host-hardware subsystem it can't hold (the C11 GPU/VFIO/device detection shims + credentialHealth
+ the core install-hint/device tables, all still core, multi-caller with deploy/vm) over the generic
`HostBuild("hostprobe")` seam (charly/host_build_hostprobe.go, sdk/spec/doctor_wire.go), which returns
raw facts only. charly/doctor.go was deleted; the report types live in the plugin now.
