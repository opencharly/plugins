# 2026.189.0906 — document the gh pr merge allow-rule (enforcement lives in GitHub)

Adds a git-workflow B5 note: `.claude/settings.json` carries a `Bash(gh pr merge:*)`
allow-rule so Claude Code's auto-mode permission classifier does NOT re-litigate
each merge. The deterministic authority is the `charly/claude-validation` status
that branch protection enforces server-side (an unvalidated PR physically cannot
merge), keeping enforcement in ONE place (GitHub) instead of duplicating it in the
client classifier. Status-POSTING is deliberately NOT allow-listed, so the
independence check stays the fresh evaluator's job. Pairs with the superproject
`.claude/settings.json` rule.
