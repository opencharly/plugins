## What was stale and why

- **`plugins/internals/skills/cloud-init-renderer/SKILL.md`** — the
  "composePackages + composeRunCmd renderer defaults" section (and its
  distro-branched-delivery paragraph) described `composeRunCmd` as prepending
  a SINGLE `systemctl enable --now sshd` step, and the file-layout table
  omitted `composeBootCmd` entirely. `sdk#87` adds `composeBootCmd` (masks
  `ssh.socket` at the earliest cloud-init phase) and rewrites `composeRunCmd`
  to a three-step prepend (a self-testing `PerSourcePenalties no` sshd
  drop-in, the matching `ssh.socket` unmask, then the sshd enable) — the guest
  SSH hardening fix for the RCA'd kex-reset wedge class. The claim was
  accurate BEFORE that PR; the divergence is that PR's own surface, so the
  sweep is claim-keyed and lands in the SAME batch (R2), not a follow-up.
  Also corrected a now-stale "byte-identical to before" claim in the D15
  pacman-family paragraph — every distro's `runcmd:` is now longer, not just
  the pacman-family one.
- **`plugins/vm/skills/vm/SKILL.md`** — the "User-Level Defaults" section
  documented only `CHARLY_VM_BACKEND`. `sdk#87` adds
  `vmshared.VmStateRoot()` / `CHARLY_VM_STATE_DIR`, the worktree-scoping
  override for the VM host-state root directory (closing the "global,
  non-worktree-scoped VM state dir" footgun). Documented as its own paragraph
  (env-only, no `charly settings` key, so it doesn't fit the settings-backed
  table above it).
