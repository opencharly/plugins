# 2026.189.0720 — standing order: ALL wire types are CUE-sourced (SDD), exceptions RDD-verified

Documents the operator standing order in the SDD-operationalization owner
(`/charly-internals:go` "Generation coverage"): **ALL wire types — the
`sdk/spec/*_wire.go` host↔plugin / render-context data-carrier structs + every
plugin's params — are CUE-sourced in `sdk/schema/*.cue` and generated by
`task cue:gen`; hand-writing a wire struct is FORBIDDEN.** A wire type is a plain
or DISCRIMINATED struct, which `cue exp gengotypes` generates faithfully
(RDD-proven live). **No `@go(-)` / hand-written type is added without a full RCA +
a live gengotypes spike proving CUE genuinely cannot express it** — an unverified
exception is a mandate violation.

`cue exp gengotypes` run, not assumed): authored-config DISJUNCTIONS
(`union_types.go` — `{cloud_image}|{bootc}` → `map[string]any`); open-tailed
struct+map state (`hand_state_types.go` — `{known, [string]:_}` → `map[string]any`);
and the gRPC TRANSPORT schema (`proto` — no `cue→proto` generator exists, CUE has
no gRPC-service concept, and it carries the CUE types as `bytes *_json` envelopes,
duplicating no spec type). Reframes the former deploy_wire "classified candidate
for future schema-sourcing" as a conversion-in-progress under the mandate. Pairs
with the CLAUDE.md SDD-pillar mandate (superproject).
