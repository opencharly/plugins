**`internals/skills/agents/SKILL.md`:**

- New subsection **"The north-star protocol — align a multi-teammate program BY
  CONSTRUCTION"** (beside the orchestrator architectural-integrity duty + the
  responsibility matrix): before fanning a multi-teammate program out, the
  orchestrator AUTHORS a binding NORTH-STAR document with four parts — (a) the
  concrete END-STATE (tables + call paths, never aspirations), (b) ORDERED
  decision heuristics for the hard calls (new-seam? → check the shared data spine,
  register the IOU + STOP-and-ask, never build a seam a later wave deletes;
  needs-forbidden-types? → the move WAITS, register the IOU, never fake/move/alias;
  bodies move, shells follow; placement by DOMAIN not filename; when in doubt
  STOP-and-ask), (c) the OBSERVED ANTI-PATTERNS list, (d) the current MEASURED
  state. The document travels with every spawn BY PATH; a task-vs-north-star
  conflict is STOP-and-ask, never a local resolution. Rationale: scoped executors
  cannot make whole-board decisions, so alignment is achieved by construction.
- The same subsection adds two COMPANION INSTRUMENTS: the **IOU REGISTER** (every
  deferred "until-wave-N" item is an entry carrying size + enabler + delivering
  wave; no wave closes holding an unregistered IOU; the next wave's plan is built
  FROM the register — the whole-program form of R2's no-follow-up-someday), and
  **PER-MERGE MEASUREMENT** (the orchestrator measures the program's tracked metric
  at every merge against the plan; a >20% variance is explained + the residual
  reassigned to a named wave, never silently eroded). Its sharpest check is the
  **measured-delta rule** — a relocation whose measured delta is dispatch-shells
  while the engine stays behind a reentry seam is REJECTED at orchestrator review:
  bodies move, shells follow.

**`internals/skills/git-workflow/SKILL.md`:**

- The "Concurrent landings" BEHIND-branch recovery gains one tight paragraph: a
  gitlink ANCESTOR bump (one side an ancestor of the other) makes `gh pr
  update-branch` conservatively flag the PR CONFLICTING rather than
  fast-forwarding the gitlink to the descendant. The compliant recovery is the
  update-branch EQUIVALENT done LOCALLY — `git merge origin/main` in the feat
  worktree (git resolves the gitlink to the descendant automatically), pushed
  FAST-FORWARD; a merge, never a rebase, no force-push.
