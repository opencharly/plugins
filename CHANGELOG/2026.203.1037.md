# 2026.203.1099 — docs/lessons batch: pr-validator excuse catalog + model-identity source + B7 worktree/bed-collision lessons

Docs-only thematic batch collecting three accumulated, verified lessons from

## What changed

- **`internals/agents/pr-validator.md` — the excuse catalog (operator
  directive: validators must call out "bullshit excuses").** Added four
  sub-findings to the concurrency/forbidden-framing anti-cheat checklist item:
  (a) proven-dead code inside a PR's own touched modules is BLOCKING even when
  framed as "not this PR's problem"; (b) a parking phrase ("pre-existing",
  "out of scope", "tracked debt", "conversion-in-progress", …) is itself a
  finding unless the PR names a concrete exit it visibly advances; (c)
  modifying a non-compliant surface (a hand-written wire type, an alias)
  obligates curing it in the same PR, not merely extending it — citing the
  landed `opencharly/sdk#85` `clean_wire` precedent, generalized by standing
  operator directive 2026-07-22 and first applied (still in flight as of this
  writing) to `sdk/spec/arbiter_wire.go` in `opencharly/sdk#90`; (d) every
  sweep/dead-code claim requires its actually-executed command output pasted,
  never a
  described-but-unrun claim — citing the verified `opencharly/charly#175`
  round-1 incident (a pasted `git grep` claimed CHANGELOG-only hits while a
  fresh re-run found 60+ live source hits). The checklist's Output Format
  one-liner for this item was extended to name all four sub-findings.
- **`internals/agents/pr-validator.md` — model-identity source sharpening.**
  The existing "verify the harness/model against runtime evidence"
  instruction now NAMES the authoritative source explicitly — the validator's
  OWN session transcript
  (`~/.claude/projects/<project>/$CLAUDE_CODE_SESSION_ID.jsonl`, the
  assistant-turn `message.model` field) — and explicitly warns off
  `~/.claude/settings.json`, which holds the MAIN session's configured
  default and was the proven-wrong source in a live misattribution incident
  (a `model:sonnet`-spawned validator footer-claimed "Fable 5" by reading
  settings.json; its own transcript showed `claude-sonnet-5` throughout).
  Applied at both the commit-trailer verification item and the PR-comment
  attribution instruction.
- **`internals/skills/git-workflow/SKILL.md` (B7, worktree refresh) +
  `internals/skills/agents/SKILL.md` (bed-ownership rules) — the B7
  worktree/bed-collision lessons.** Folded in:
  - A disposable `localpkg` bed builds from the submodule's on-disk
    working-tree checkout, not the committed gitlink alone — a clean gitlink
    auto-merge does not itself check out the new submodule content, so a
    post-merge worktree refresh needs its own `submodule update --checkout`
    or the bed silently builds stale source (signature: `/tmp`-derived repo
    root + a stale baked `pkgver`).
  - A derived `pkgver` left dirty in `pkg/arch/PKGBUILD` by `calver.sh`
    persists across the session, per worktree, and must be discarded before
    every bed re-gate, not only in the main checkout.
  - The per-bed-name mutex (`charly-<bed>` deploy names) is
    worktree-independent: the SAME bed name run concurrently from two
    different worktrees collides on the one deterministic domain/container
    name — including when a `pr-validator` legitimately re-runs a bed under
    its own R10 authority from a different worktree than the one already
    running it. The orchestrator's bed-serialization duty now explicitly
    covers all bed launches by name across every worktree and agent, not only
    within one team's own roster.
  - A 5th, today's instance of the class: a bed-runner sub-agent
    foreground-ran a VM bed with a `timeout` in a worktree its spawner was
    simultaneously editing — combining a foreground-with-timeout violation on
    a long bed with a within-worktree self-freeze violation, producing both
    an orphaned libvirt domain and a stale-binary guard trip. Both "Handling
    a long-running bed" and "Within-worktree self-freeze" now state
    explicitly that a spawned sub-agent is not exempt from either rule merely
    because it is a different agent from its editing spawner — the worktree,
    not the agent, is the shared resource being gated.
