# pr-validator always posts a PR comment with its findings

The fresh `pr-validator` agent (`internals/agents/pr-validator.md`) now, in Phase 2,
ALWAYS posts a human-readable PR comment (`gh pr comment`) alongside the
`charly/claude-validation` commit status — on BOTH pass and fail — stating the
per-item checklist verdict and WHY the PR is or is not approved (the specific
blocking findings + file:line on a fail). Previously the reasoning lived only in
the commit-status description + the delegating session's paste; now it is visible
on the PR itself for any human reader. On a re-run after a fix the evaluator posts
a fresh comment.
