# 2026.190.2350 — `/verify-beds` runs only what a sub-agent can own (doctrine sweep)

The `/verify-beds` workflow drove **every** disposable bed through an ephemeral `agent()`
sub-agent. A sub-agent returns synchronously, its background children die with it, and its one
foreground `charly check run` is capped by the Bash tool at 600s — so it **cannot own a bed
that outlives its turn**. A roster run consequently returned 26 real passes and **26 beds with
no verdict**, orphaned 3 libvirt domains and 4 pod containers, recorded a bed that had actually
passed as a failure, and ran a `host: local` deploy that installed packages as root on the
operator's workstation.

`/charly-internals:agents` already forbade it (*"NEVER the sub-agent `/verify-beds` workflow
for >600s beds"*) — but four other surfaces still described the old contract. This is the
claim-keyed R1 sweep of those surfaces; the workflow itself and CLAUDE.md change in the
superproject cutover this rides with.

**The workflow's real contract now.** Beds partition three ways, and nothing is silently
dropped:

- **SHORT** — run by the workflow, in parallel, as before.
- **LONG** — `vm`/`android` substrate, or a bed whose newest `.check/<bed>/<calver>/summary.yml`
  records `total_seconds >= 600`. **Deferred, not run**, and returned in `deferredLongBeds[]`
  with the exact command; the PERSISTENT session owns each as a `run_in_background` task.
- **HOST-LOCAL** — a `local:` deploy, or a bed with a nested `local:` member, whose `host:` is
  `local`. **Refused, not run**: it applies candies to the operator's workstation, and test
  beds belong in a disposable eval VM.

The result carries `gateComplete`, so a partial roster can no longer read as a green R10 gate.

**Surfaces corrected (each asserted that `/verify-beds` runs every bed):**

- `/charly-internals:agents` — the workflow roster entry now states defer/refuse/`gateComplete`.
- `/charly-check:check` — the "whole roster" instruction, the flag-discipline parenthetical, the
  wall-clock section, and the "R10 gate by change class" matrix row for `charly` Go code.
- `/charly-internals:go-quality` — the R10-gate cross-reference for tree-wide categories.
- `internals/agents/check-bed-runner.md` — "one instance per bed" → one per SHORT bed.

**Also recorded (a trap that cost a near-repeat of the incident):** `Workflow({name})` resolves
a registry snapshot taken at session start, so an edited `.claude/workflows/*.js` is **not**
what runs. Drive an edited workflow with `Workflow({scriptPath: …})`, which takes precedence.
