# 2026.197.0744 — Validator provisioning ledger

The PR-validator policy now makes protected-main bootstrap a parent-owned, immutable
handoff. Before each fresh validation round, the parent runs
`task agent:prepare-validator-worktree` with an absolute superproject path, the exact
40-character protected `origin/main` SHA, and a new empty worktree path. The task's
`scripts/prepare-validator-worktree.py` implementation creates the detached tree,
recursively initializes submodules, verifies the validator specification and both
Claude/Codex developer setup checks, and emits the worktree/base/submodule/check ledger.

The fresh validator verifies that ledger with read-only filesystem reads before R0, then
uses only the sealed tree. It may not initialize submodules, run setup, create a
worktree, or repair bootstrap state itself. A provisioning defect blocks validation and
requires a new freshly provisioned validator round, preventing a partial plugin checkout
from being treated as a readable policy baseline.
