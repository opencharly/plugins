## What changed upstream (the sdk PR this sweep describes)

The remaining 13 hand-written `sdk/spec/*_wire.go` files — `agent`, `deploy`
(the biggest, 987 lines), `distro`, `doctor`, `enc`, `feature`, `gpu`, `init`,
`k8sgen`, `resource`, `settings`, `substrate_template`, `vm` — are converted
to CUE defs per the standing SDD mandate ("WIRE TYPES ARE CUE-SOURCED WITHOUT
EXCEPTION"). Every hand-written wire struct file is now DELETED from
`sdk/spec`; `sdk/spec/*_wire.go` is an EXTINCT file class. Two reshapes were
required for byte-compatible generation:

- `HostProbeReply.GroupAccessible` (doctor) — `map[int]bool` → `map[string]bool`
  (CUE has no int-keyed-map construct; `encoding/json` already renders a
  `map[int]bool`'s keys as decimal strings on the wire, so this is a pure
  representation fix, zero wire-format change). Two charly-side call sites
  updated (`charly/host_build_hostprobe.go`, `candy/plugin-doctor/command.go`).
- `VFIOGpu` (gpu) — the former anonymous-embedded `VFIOPCIDevice` FLATTENED
  (CUE unification has no Go-embedding concept; Go's own JSON-promotion of an
  embedded field already made the wire shape identical either way). A new
  `spec.NewVFIOGpu(d)` constructor (`spec/gpu_methods.go`) replaces every
  `VFIOGpu{VFIOPCIDevice: d}` composite literal (candy/plugin-gpu,
  charly test fixtures).

`Scope` (an int enum + `String()`/`ScopeFromName`) and `ReverseOpKind` (a
string enum) stay hand-written (`spec/deploy_consts.go`), mirroring the
`Status` / `status_result.go` precedent — gengotypes has no construct for an
iota-enum + Stringer, and a plain string-enum never carries a JSON/YAML shape
to generate. `Diagnostics.HasErrors()`, `LifecycleOptsFromEmit`,
`ResolvedDistro.PrimaryFormat()`/`LocalPkgFormat()`, and
`ResolvedAndroid.IsEndpoint()`/`EffectiveSerial()` are pure Go methods/functions
CUE cannot express — hand-written beside their CUE-generated struct siblings
(`spec/deploy_methods.go`, `spec/distro_methods.go`,
`spec/substrate_template_methods.go`).

A `#KeyValue` def name was RDD-caught colliding with the schemagen
`kindValueDefs` auto-registration (any top-level `#<X>Value` def is treated
as a kind-value gate) — renamed to `#EnvKV` with a `KeyValue = EnvKV`
charly-name alias (`spec/charly_names.go`) preserving every real consumer's
type name unchanged.
