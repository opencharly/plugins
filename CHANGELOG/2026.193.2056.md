**`internals/skills/agents/SKILL.md`:**

- The hooks-doctrine attribution-absence description said "a `-F`/heredoc/command-substituted message
  is not scanned for absence". That was the fail-OPEN hole: the tier came ONLY from the command
  string, so an external `-F <file>` (or a `$(...)` substitution) hid it and every tier-dependent
  check silently passed. Corrected to: the trailer is read from a READABLE message — an inline `-m`
  value, a heredoc body (both in the command string), or a `-F <file>` the gate now READS — and a
  message the gate CANNOT read (a `$(...)`/backtick substitution, a piped/unreadable `-F`, or an
  editor message with no -m/-F) fails CLOSED; `--amend`/reuse inherit an already-gated message and
  are exempt. (RDD note: a `<<` heredoc body IS in the command string, so heredoc was never a hole —
  only external files and substitutions were.)
- The CHANGELOG-check firing condition "fires only when an inline tier is parsed" → "fires whenever a
  tier is parsed from a READABLE message (inline `-m`, heredoc, or a `-F <file>`)".
