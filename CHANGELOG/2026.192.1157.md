# 2026.192.1157 — context-exhaustion is a forbidden excuse; "fresh context" is spelled DELEGATE

Close the last stop-excuse loophole and make the autonomous-loop + delegation doctrine crystal clear across
the discipline skills (paired with the CLAUDE.md + hooks edits in the superproject).

- **`cutover-policy`** — the forbidden-internal-voice trigger list gains the CONTEXT framings ("I've run out
  of / exhausted context", "continuing would leave a broken, unbuildable tree" — a FABRICATED excuse with
  ZERO evidence, never raised, "this needs a fresh session / fresh context", "the correct next step is a
  fresh session"), plus an explicit clarification: **"fresh context / a fresh session" is NEVER a stop — it
  is spelled DELEGATE.** When a cutover (or a whole multi-cutover program) exceeds the current context, you
  compact-and-continue OR spawn a fresh teammate / sub-agent that carries the unit with a full fresh context
  budget; that teammate IS "a fresh session," delivered on demand inside the same autonomous run.
- **`strict-policy`** — the R2 forbidden-excuse catalog's size framings gain the same CONTEXT framings +
  the delegate-don't-stop rule.
- **`agents`** — a new "Delegation IS fresh context — the primitives' primary purpose" section: a
  sub-agent/teammate/workflow runs with its own fresh context budget, so delegation is the ANSWER to context
  pressure, never a reason to stop; a large program is driven a teammate per cutover to merged PRs, one clean
  atomic cutover per teammate, orchestrated by the persistent session.

Pairs with the superproject CLAUDE.md ("Hard Cutover by Default" Size/budget/context bullet + "Agents,
Workflows & Teams" delegation mandate) and the `runtime-verification-reminder` / `end-of-turn-challenge`
hook pointers.
