# go skill: preempt proxy row — command:preempt reaches verb:arbiter via InvokeProvider; HostArbiter always-served

`charly preempt` (command:preempt) no longer shells back through hidden `__preempt-*` core commands +
the in-core proxy. It is COMPILED-IN and reaches its peer verb:arbiter DIRECTLY via InvokeProvider,
owning the lease-table formatting itself. The in-core arbiterProxy stays (slimmed — Status/reconcileStranded
removed) as the generic core→verb registry bridge the core lease-lifecycle callers use (core is not a
plugin, so it cannot call InvokeProvider). HostArbiter is now an always-served generic reverse leg
(stateless, inert on non-arbiter channels) so command:preempt → InvokeProvider → verb:arbiter → HostArbiter
round-trips over the bare command-dispatch executor.
