# Sweep skill references to the retired check-verb preresolver architecture (R5/R1, H part 2)

The H-part-2 endpoint-seam migrations (upstream v2026.187.2110–2319) deleted the per-verb host
preresolvers (preresolveCdpEndpoint / preresolveVncEndpoint / preresolveSpiceEndpoint /
preresolveMcpEndpoint / preresolveKubeCluster), the verbPreresolver registry, and the
CheckEnv.Substrate channel — every check verb now resolves through the class-generic
CheckContextService reverse-legs (cc.ResolveEndpoint / ResolveGraphicsEndpoint /
ResolveClusterContext / ResolveImageLabel). Five skills still described the old host-preresolver
architecture as current — R1 makes that documentation divergence an incident, swept claim-keyed:

- `internals/skills/go` — the architecture-map rows (verb_preresolve.go / cdp_preresolve.go /
  vnc_preresolve.go / mcp_preresolve.go) rewritten to check_endpoint_resolve.go's reverse-legs.
- `check/skills/spice`, `check/skills/vnc` — the host pre-resolves → the plugin resolves via
  the reverse-leg; spice_preresolve.go/vnc_preresolve.go → resolveVerbGraphics + vnc_helpers.go.
- `build/skills/charly-mcp-cmd` — the host reads the mcp_provide label → the plugin reads it via
  cc.ResolveImageLabel + maps the port via cc.ResolveEndpoint.
- `kubernetes/skills/check-k8s` — preresolveKubeCluster → the cc.ResolveClusterContext leg.

Documentation-only (skill prose), zero behaviour change.
